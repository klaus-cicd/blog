Hexo安装

  1. Git
    1. Git (git-scm.com)
  2. NodeJS
    [[安装Node]]

初始化

  1. hexo安装（-g为golbal缩写，表示全局安装）

  2. 在本地指定目录下创建文件夹，比如叫“hexo-blog”
    在“hexo-blog”下打开命令行，执行

  3. 安装 hexo 依赖包

启动

  • 执行成功后默认地址为：http://localhost:4000
  • 对于主题的修改，支持热更新，在启动状态下刷新页面即可生效
  • 对于hexo的配置修改，需要停止后重新启动才可生效

新增文章

默认有一篇helloworld文章，至少要有一篇文章才能进入首页

在hexo目录下执行hexo new post xxx，xxx替换为你想要的文章名称，文章创建后会自动在hexo/source/_post目录下创建一个xxx.md文件

配置

语言修改

  • 改为英文
    默认就是英文 en
  • 改为简体中文
    在Hexo根目录下找到_config.yml文件，修改该文件的language标签， 改为：language: zh-CN

开启文章图片共存

开启后每次使用hexo n post xxx新增文章时，会在文章同级目录下创建一个同名文件夹，该文章内的图片即可放入该文件夹内

主题

主题选择

Hexo拥有很多主题，都在Themes | Hexo可以找到
其中使用最多的为NexT主题，主打一个简约，主题地址：https://github.com/next-theme/hexo-theme-next

主题配置

  1. 下载/安装
  • 方式一

在Hexo的themes目录下执行：git clone https://github.com/next-theme/hexo-theme-next themes/next
通过该方式下载的主题存放于hexo/themes内

  • 方式二

    对于Hexo 5.0以上的版本，可以直接在Hexo根目录下安装主题：npm install hexo-theme-next
    通过该方式安装的主题，存放于hexo/node_modules目录下，修改主题配置时要注意

  1. 主题更换

打开在博客根目录下的_config.yml 文件，找到theme配置，将值改为next（即在themes目录下主题文件夹的名称）

  1. 主题升级

如果是通过Git clone的主题，则拉取最新主题代码即可

生成文章唯一链接

Hexo在生成拨款了文章链接时，默认是按照年月日标题格式生成，所以如果链接被分享后对文章标题进行修改，使用旧的链接就无法正常访问到文章了

可以利用hexo-abbrlink插件来生成文章唯一链接从而解决这个问题（原理：生成后原文章md文件的Front-matter内会增加abbrlink字段，这个id就会作为文章的专属链接Î）

  1. 在hexo根目录下执行插件安装指令：npm install hexo-abbrlink --save
  2. 修改配置文件

创建一个公开的git仓库（一定要为公开的）

申请Gitee Pages服务

  1. 创建仓库后，在服务Tab内找到Gitee Pages （服务Tab下的第一个菜单）

  1. 首次申请时需要进行实名认证，自行提交认证所需材料，一般1至2个工作日申请OK

  2. 在本地Hexo根目录配置文件_config.yml最下方找到deploy相关内容，填入你的Git仓库地址信息

    自行替换your-git-name和your-repo-name部分（在Git仓库，代码Tab下，点击克隆/下载按钮可快速复制你的Git仓库地址）

    branch可不填，默认为master

  3. 在本地Hexo根目录下打开命令行，执行以下内容

    打开Git仓库查看本地的Hexo相关内容是否已经推送至Git仓库

Github page

这里介绍一键部署方式，可以避免将源码公示

参考：

https://hexo.io/zh-cn/docs/github-pages

https://github.com/hexojs/hexo-deployer-git

  1. 安装插件：npm install hexo-deployer-git --save
  2. 建立一个Github仓库，对于免费版的Github Page，
    1. 要求仓库必须是公开的，付费后将仓库改为私有
    2. 仓库名称如果不是<username>.github.io，需要在仓库Pages设置内自定义别名，但是一定要以.github.io结尾
    3. 由于部署时只会将生成的html文件推至配置的分支，所以建议使用两个分支，一个用来管理源码，一个用于部署
  3. 修改Hexo配置

或

  1. 将Hexo推送至上面配置的github仓库内

    这里使用main作为主分支，执行git init后默认是使用master作为主分支，如果需要改成main，可以再执行git branch -m main 对分支进行重命名

  2. 部署：hexo clean && hexo deploy
  3. 提示部署成功后访问 <GitHub用户名>.github.io，检查是否ok
